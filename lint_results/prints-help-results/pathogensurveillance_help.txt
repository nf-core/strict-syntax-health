Input/output options
  --input                       [string]  Path to comma/tab-separated file containing information about samples. 
  --max_parallel_downloads      [integer] Maximum number of parallel downloads allowed for certain processes. This can be set to control the 
number of concurrent downloads to avoid overloading servers or hitting API rate limits.  
  --reference_data              [string]  Path to comma-separated file containing information about references. 
  --outdir                      [string]  The output directory where the results will be saved. You have to use absolute paths to storage if 
running on Cloud infrastructure.  
  --temp_dir                    [string]  The location to save temporary files for processes. This is only used for some processes that 
produce large temporary files such as PICARD_SORTSAM.  
  --data_dir                    [string]  The location to save downloaded files for later use. This is seperate from the cached data (usually 
stored in the 'work' directory), so that the cache can be cleared without having to repeat many 
large downloads. [default: path_surveil_data]  
  --email                       [string]  Email address for completion summary. 
  --multiqc_title               [string]  MultiQC report title. Printed as page header, used for filename if not otherwise specified. 
  --bakta_db                    [string]  The path to the Bakta database folder. This or --download_bakta_db must be included. 
  --download_bakta_db           [boolean] Download the database required for running Bakta. This or --bakta_db must be included. For more 
information, see: https://github.com/oschwengers/bakta?tab=readme-ov-file#database-download 
[default: true]  
  --bakta_db_type               [string]  Which type of the Bakta database to download. Must be 'light' (~2Gb) or 'full' (~40Gb). 
[default: light]  

Analysis parameters
  --max_samples                 [integer]         Maximum number of samples to analyze. The first n samples will be used. [default: 1000] 
  --max_depth                   [number]          Maximum depth of reads to be used for all analyses. Samples with more reads are subsampled to this 
depth. [default: 100]  
  --only_latin_binomial_refs    [boolean]         When selecting references automatically, only consider references with names that appear to be 
standard latin bionomials (i.e. no numbers or symbols in the first two words). [default: false] 
 
  --allow_non_refseq            [boolean]         When selecting references automatically, allow assemblies not in RefSeq. [default: false] 
  --allow_unannotated           [boolean]         When selecting references automatically, allow assemblies that do not have annotations. The pipeline 
will attempt to annotate these references as needed. [default: false]  
  --allow_atypical_refs         [boolean]         When selecting references automatically, allow assemblies that NCBI considers atypical. 
[default: false]  
  --n_ref_strains               [integer, string] The maximum number/percentage of references representing unique subspecies to download from RefSeq 
for each species predicited to be in each sample. Samples with similar initial indentifications will 
usually use the same references, so the total number of references downloaded for a goup of samples 
will depend on the taxonomic diversity of the samples. [default: 5]  
  --n_ref_species               [integer, string] The maximum number/percentage of references representing unique species to download from RefSeq for 
each genus predicited to be in each sample. Samples with similar initial indentifications will 
usually use the same references, so the total number of references downloaded for a goup of samples 
will depend on the taxonomic diversity of the samples. [default: 20]  
  --n_ref_genera                [integer, string] The maximum number/percentage of references representing unique genera to download from RefSeq for 
each family predicited to be in each sample. Samples with similar initial indentifications will 
usually use the same references, so the total number of references downloaded for a goup of samples 
will depend on the taxonomic diversity of the samples. [default: 20]  
  --n_ref_closest               [number]          The number of references most similar to each sample based on estimated ANI to include in 
phyogenetic anlyses. [default: 2]  
  --n_ref_closest_named         [number]          Same as the 'n_ref_closest' option except that it only applies to referneces with what apppear to be 
standard latin binomaial names (i.e. two words with no numbers or symbols). This is intended to 
ensure that a refernece with an informative name is present even if it is not the most similar. 
[default: 1]  
  --n_ref_context               [number]          The number of references representing the entire range of ANI relative to each sample. These are 
meant to provide context for more similar references. For a group of samples, the fewest total 
references will be selected that satisify this count for each sample. [default: 5]  
  --phylo_min_genes             [number]          The minimum number of genes needed to conduct a core gene phylogeny. Samples and references will be 
removed (as allowed by the `min_core_samps` and `min_core_refs` options) until this minimum is met. 
[default: 10]  
  --phylo_max_genes             [number]          The maximum number of genes used to conduct a core gene phylogeny. [default: 500] 
  --ref_min_ani                 [number]          The minimum ANI between a sample and potential reference for that reference to be used for mapping 
reads from that sample. To force all the samples in a report group to use the same reference, set 
this value very low. [default: 0.95]  
  --min_bases_to_assemble       [number]          The minimum number of bases after quality filtering needed to attempt an genome assembly. Samples 
with few than this amount of bases will be excluded from further analysis. [default: 100000] 
 
  --max_variants                [number]          The maximum number of variants to use for the SNP tree and minimum spanning nextwork produced by the 
variant analysis. The first N variants are used when more than N are available. [default: 
100000]  
  --cpu_scale                   [number]          A scaling factor to adjust the number of CPUs used by multithreaded processes. The number of CPUs 
for each process will be multiplied by this value and rounded to the nearest integer (minimum 1). 
[default: 1.0]  
  --skip_core_phylogeny         [boolean]         Skip the core genome phylogeny analysis. This will not affect downstream processes like MultiQC and 
the main report. [default: false]

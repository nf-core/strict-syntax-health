Typical pipeline command:

  nextflow run nf-core/seqsubmit -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Input/output options
  --input                       [string] Path to comma-separated file describing the data to be submitted. Format depends on the pipeline 
mode (mags/bins/metagenomic_assemblies/reads).  
  --outdir                      [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                       [string] Email address for completion summary. 
  --multiqc_title               [string] MultiQC report title. Printed as page header, used for filename if not otherwise specified. 

RNA detection options
  --rrna_limit                  [number] Minimum percentage of 16S, 23S, and 5S rRNA gene length recovered to count the gene as present. 
[default: 80]  
  --trna_limit                  [number] Minimum number of tRNA. [default: 18] 

Assigning NCBI taxonomy with CAT_pack
  --cat_db                      [string] Path to local CAT_pack DB 
  --cat_db_download_id          [string] CAT_pack database ID for download [default: nr] 

Evaluation options
  --checkm2_db                  [string]  Path to pre-downloaded checkm2 database 
  --checkm2_db_download_id      [integer] Zenodo ID for checkm2 database download [default: 14897628] 

Generic options
  --multiqc_methods_description [string]          Custom MultiQC yaml file containing HTML including a methods description. 
  --help                        [boolean, string] Display the help message. 
  --help_full                   [boolean]         Display the full detailed help message. 
  --show_hidden                 [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided). 
 

Pipeline flow control parameters
  --centre_name                 [string]  The center name of the submitter (mandatory for broker accounts). 
  --upload_tpa                  [boolean] Flag to control the type of assembly study (third party assembly or not) [default: false] 
  --mode                        [string]  Type of upload  (accepted: mags, bins, metagenomic_assemblies, reads) 
  --test_upload                 [boolean] Upload to TEST server instead of LIVE [default: true] 
  --is_private                  [boolean] Use that flag if you are referring to private data in ENA [default: false] 
  --release_date                [string]  Use that flag if you want to keep your data private after submission 
  --submission_study            [string]  ENA study accession (PRJ/ERP) to submit the data to 
  --study_metadata              [string]  Path to study metadata file (JSON, CSV, or TSV) for registering a new ENA study. Required when 
submission_study is not provided.  
  --webincli_mode               [string]  Webin-CLI mode of operation  (accepted: submit, validate) [default: submit]

Typical pipeline command:

  nextflow run nf-core/multiplesequencealign -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Basic Input/output options
  --input                       [string] Path to the samplesheet file containing information about datasets to be aligned and evaluated 
(samplesheet).  
  --outdir                      [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --seqs                        [string] Path to the input fasta file. 
  --pdbs_dir                    [string] Path to a folder containing all the optional data files to be used (e.g. structures). 
  --templates_suffix            [string] Suffix of the files given in the optional_data directory. [default: .pdb] 
  --email                       [string] Email address for completion summary. 
  --multiqc_title               [string] MultiQC report title. Printed as page header, used for filename if not otherwise specified. 

Tools input options
  --tools                       [string] Path to the file containing information about the tools to be deployed (toolsheet). 
  --tree                        [string] Tool to use for tree building.  (accepted: FAMSA, CLUSTALO, MAFFT) 
  --args_tree                   [string] Extra arguments for the tree building tool. 
  --aligner                     [string] The aligner to be used.  (accepted: CLUSTALO, FAMSA, KALIGN, LEARNMSA, MAFFT, MAGUS, MUSCLE5, 
TCOFFEE, REGRESSIVE, UPP, 3DCOFFEE, ...)  
  --args_aligner                [string] Extra arguments for the aligner tool. 

Global options
  --use_gpu                     [boolean] Run on CPUs (default) or GPUs 
  --skip_compression            [boolean] Produce uncompressed alignment files 
  --skip_preprocessing          [boolean] Skip the preprocessing step for the input files. 
  --skip_validation             [boolean] Skip the validation of the input files. 
  --skip_pdbconversion          [boolean] Skip the conversion of pdb files to fasta. 

Align options
  --build_consensus             [boolean] Build consensus alignment with M-COFFEE. 

Stats options
  --skip_stats                  [boolean] Skip all of the statistics computation on the fasta file. 
  --calc_sim                    [boolean] Calculate the percentage similarity across the sequences in the input files. 
  --calc_seq_stats              [boolean] Calculate general statistics on input files. 
  --extract_plddt               [boolean] Extract plddt from structure files. Only works if the files have been generated by AF2. 

Eval options
  --skip_eval                   [boolean] Skip all of the evaluations computation on the msa. 
  --calc_sp                     [boolean] Calculate the Sum of Pairs of alignment. [default: true] 
  --calc_tc                     [boolean] Calculate the Total Column Score of alignment. [default: true] 
  --calc_irmsd                  [boolean] Calculate the iRMSD of alignment. 
  --calc_tcs                    [boolean] Calculate the Transitive Consistency Score (TCS) of alignment. 
  --calc_gaps                   [boolean] Extract total number of gaps and average number of gaps of the alignment. [default: true] 

Reports options
  --skip_multiqc                [boolean] Skip the multiqc report generation. 
  --skip_shiny                  [boolean] Skip the shiny report generation. 
  --shiny_app                   [string]  Folder containing the main shiny app. 
  --skip_visualisation          [boolean] Skip the visualization generation. 

Generic options
  --multiqc_methods_description [string]          Custom MultiQC yaml file containing HTML including a methods description. 
  --help                        [boolean, string] Display the help message. 
  --help_full                   [boolean]         Display the full detailed help message. 
  --show_hidden                 [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).

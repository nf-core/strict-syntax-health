Typical pipeline command:

  nextflow run nf-core/mcmicro -profile <docker/singularity/.../institute> [--input_cycle|--input_sample] samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Input/output options
  --input_sample                [string]  Path to comma-separated file containing information about the samples in the experiment. 
  --input_cycle                 [string]  Path to comma-separated file containing information about the samples in the experiment. 
  --outdir                      [string]  The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                       [string]  Email address for completion summary. 
  --multiqc_title               [string]  MultiQC report title. Printed as page header, used for filename if not otherwise specified. 
  --marker_sheet                [string]  Path to comma-separated file containing information about the markers 
  --illumination                [string]  input that defines type of illumination correction to be performed  (accepted: basicpy) 
  --tma_dearray                 [boolean] If the sample is a tissue microarray (TMA), set this to 'true' to generate a separate output for 
each tissue core for pipeline steps beginning with cell segmentation. This also tends to speed up 
processing and reduce RAM usage.  
  --segmentation                [string]  comma separated list of segmentation modules to run. options are ['mccellpose','cellpose','mesmer'] 
 
  --cellpose_model              [string]  optional model file for cellpose segmentation 
  --backsub                     [boolean] boolean to flag whether or not to apply background subtraction 
  --prelude                     [boolean] Run the pipeline in prelude mode. Extracts metadata writes MultiQC table and exits. 

Generic options
  --multiqc_methods_description [string]          Custom MultiQC yaml file containing HTML including a methods description. 
  --help                        [boolean, string] Display the help message. 
  --help_full                   [boolean]         Display the full detailed help message. 
  --show_hidden                 [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).

Typical pipeline command:

  nextflow run nf-core/variantprioritization -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Input/output options
  --input                           [string] Path to comma-separated file containing information about the samples in the experiment. 
  --outdir                          [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                           [string] Email address for completion summary. 
  --multiqc_title                   [string] MultiQC report title. Printed as page header, used for filename if not otherwise specified. 

Reference options
  --genome                          [string]  Name of iGenomes reference. 
  --pcgr_bundleversion              [integer] PCGR/CPSR reference data bundle version. [default: 20250314] 
  --pcgr_database_dir               [string]  Path to PCGR database. Set `pcgr_download` to `false` when using this. 
  --pcgr_download                   [boolean] Download PCGR reference database from the PCGR site. Set to `false` if you want to use a local path. 
[default: true]  
  --vep_cache                       [string]  Path to VEP cache. [default: s3://annotation-cache/vep_cache/] 
  --vep_cache_version               [integer] VEP cache version. [default: 113] 

CPSR options
  --panel_id                        [string]  Comma-separated CPSR virtual panel identifier(s). [default: 0] 
  --custom_list                     [string]  Path to custom gene list file (single-column Ensembl gene identifiers). 
  --custom_list_name                [string]  Display name for custom panel/list. 
  --diagnostic_grade_only           [boolean] For PanelApp panels (1-44), include GREEN-status genes only. 
  --secondary_findings              [boolean] Include variants in ACMG secondary findings gene list (v3.2). 
  --pgx_findings                    [boolean] Report overlap with pharmacogenomic toxicity variants (CPIC/PgX). 
  --gwas_findings                   [boolean] Report overlap with low/moderate-risk GWAS cancer variants. 
  --pop_gnomad                      [string]  gnomAD population source used for ACMG frequency assessment.  (accepted: afr, amr, eas, sas, 
asj, nfe, fin, global) [default: nfe]  
  --maf_upper_threshold             [number]  Upper gnomAD global MAF threshold for variants included in report. [default: 0.9] 
  --classify_all                    [boolean] Also provide CPSR TIER classifications for variants with existing ClinVar classifications. 
  --clinvar_report_noncancer        [boolean] Include ClinVar variants linked to non-cancer phenotypes/conditions. 
  --ignore_noncoding                [boolean] Ignore non-coding (non protein-altering) variants in CPSR report. 
  --vep_gencode_basic               [boolean] Consider basic GENCODE transcript set only with Variant Effect Predictor (VEP) (option 
'--gencode_basic' in VEP).  

PCGR options
  --vcf2maf                         [boolean] Generate a MAF file for the input VCF using vcf2maf. 
  --assay                           [string]  Type of DNA sequencing assay performed for input data.  (accepted: WGS, WES, TARGETED) 
  --tumor_site                      [integer] Primary tumor type/site code used for site-specific interpretation. [default: 0] 
  --tumor_ploidy                    [string]  Estimated tumor ploidy. 
  --cna_analysis                    [boolean] Enable copy-number alteration (CNA) analysis and reporting. 
  --cna_overlap_pct                 [integer] Mean percent overlap between CNA segment and gene transcripts for reporting gains/losses. 
[default: 50]  
  --tumor_dp_tag                    [string]  VCF INFO tag for tumor sequencing depth. 
  --tumor_af_tag                    [string]  VCF INFO tag for tumor variant allelic fraction. 
  --control_dp_tag                  [string]  VCF INFO tag for control/normal sequencing depth. 
  --control_af_tag                  [string]  VCF INFO tag for control/normal variant allelic fraction. 
  --call_conf_tag                   [string]  VCF INFO tag for somatic call confidence. 
  --tumor_purity                              Estimated tumor purity. 
  --effective_target_size_mb        [integer] Effective target size in Mb for TMB calculation. [default: 34] 
  --input_cpsr                      [string]  Path to CPSR-classified germline calls file. 
  --input_cpsr_yaml                 [string]  Path to CPSR YAML configuration file. 
  --estimate_tmb                    [boolean] Estimate tumor mutational burden (TMB). 
  --estimate_msi                    [boolean] Predict microsatellite instability (MSI) status. 
  --estimate_signatures             [boolean] Estimate mutational signature contributions by re-fitting. 
  --tmb_display                     [string]  TMB measure to show in report.  (accepted: coding_and_silent, coding_non_silent, missense_only) 
[default: coding_and_silent]  
  --tmb_dp_min                      [integer] Minimum tumor depth required for inclusion in TMB calculation when depth tag is available. 
  --tmb_af_min                      [number]  Minimum tumor allelic fraction required for inclusion in TMB calculation when AF tag is available. 
 
  --tmb_ad_min                      [integer] Minimum tumor allelic depth (ALT-supporting reads) required for TMB calculation when depth+AF tags 
are available.  
  --min_mutations_signatures        [integer] Minimum number of SNVs required for mutational signature re-fitting. [default: 200] 
  --all_reference_signatures        [boolean] Use all reference SBS signatures during signature re-fitting. 
  --include_artefact_signatures     [boolean] Include sequencing artefact signatures during fitting. 
  --prevalence_reference_signatures [number]  Minimum tumor-type prevalence (%) of reference signatures to include in re-fitting. [default: 
0.1]  
  --tumor_dp_min                    [integer] Minimum tumor depth for variant inclusion when tumor depth tag is provided. [default: 0] 
  --tumor_ad_min                    [integer] Minimum tumor allelic depth (ALT-supporting reads) for variant inclusion when tumor depth+AF tags 
are provided.  
  --tumor_af_min                    [integer] Minimum tumor allelic fraction for variant inclusion when tumor AF tag is provided. [default: 0] 
 
  --control_dp_min                  [integer] Minimum control/normal depth for variant inclusion when control depth tag is provided. [default: 
0]  
  --control_ad_max                  [integer] Maximum control/normal allelic depth (ALT-supporting reads) allowed when control depth+AF tags are 
provided.  
  --control_af_max                  [integer] Maximum control/normal allelic fraction allowed when control AF tag is provided. [default: 1] 
 
  --no_html                         [boolean] Do not generate PCGR HTML report. 

PCGR tumor-only options
  --tumor_only                      [boolean] Run in tumor-only mode (no matched normal sample). 
  --pon_vcf                         [string]  Path to panel-of-normals (PoN) VCF used to suppress recurrent artefacts. 
  --exclude_pon                     [boolean] Exclude variants overlapping panel-of-normals calls. 
  --exclude_likely_hom_germline     [boolean] Exclude variants likely to represent homozygous germline events. 
  --exclude_likely_het_germline     [boolean] Exclude variants likely to represent heterozygous germline events. 
  --exclude_dbsnp_nonsomatic        [boolean] Exclude variants with dbSNP support for non-somatic origin. 
  --exclude_nonexonic               [boolean] Exclude non-exonic variants in tumor-only filtering. 
  --maf_gnomad_nfe                  [number]  Maximum gnomAD NFE population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_fin                  [number]  Maximum gnomAD FIN population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_asj                  [number]  Maximum gnomAD ASJ population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_oth                  [number]  Maximum gnomAD OTH population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_amr                  [number]  Maximum gnomAD AMR population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_afr                  [number]  Maximum gnomAD AFR population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_eas                  [number]  Maximum gnomAD EAS population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_sas                  [number]  Maximum gnomAD SAS population AF tolerated in tumor-only filtering. [default: 0.002] 
  --maf_gnomad_global               [number]  Maximum global gnomAD AF tolerated in tumor-only filtering. [default: 0.002] 

Variant filtering options
  --filter_haplotypecaller          [string] Filtering expression for HaplotypeCaller VCF records. [default: -i\'FORMAT/DP>10\] 
  --filter_deepvariant              [string] Filtering expression for DeepVariant VCF records. [default: -i\'FORMAT/DP>10\] 
  --filter_strelka_variants         [string] Filtering expression for combined Strelka variant records. [default: -i\'FORMAT/DP>10\] 
  --filter_freebayes_germline       [string] Filtering expression for FreeBayes germline VCF records. [default: -i\'FORMAT/DP>10\] 
  --filter_mutect2                  [string] Filtering expression for Mutect2 VCF records. [default: -i\'FORMAT/DP>10\] 
  --filter_freebayes_somatic        [string] Filtering expression for FreeBayes somatic VCF records. [default: -i\'FORMAT/DP>10\] 
  --filter_strelka_indels           [string] Filtering expression for Strelka INDEL records. [default: -i\'FORMAT/DP>10\] 
  --filter_strelka_snvs             [string] Filtering expression for Strelka SNV records. [default: -i\'FORMAT/DP>10\] 

VEP options
  --vep_n_forks                     [integer] Number of parallel VEP forks. [default: 4] 
  --vep_buffer_size                 [integer] Number of variants buffered per VEP processing block. [default: 500] 
  --vep_gencode_all                 [boolean] Use full GENCODE transcript set during annotation. [default: true] 
  --vep_pick_order                  [string]  Priority order used by VEP when selecting a single representative consequence. [default: 
mane_select,mane_plus_clinical,canonical,biotype,ccds,rank,tsl,appris,length]  
  --vep_no_intergenic               [boolean] Skip annotation of intergenic variants. 

Generic options

  --help                            [boolean, string] Display the help message. 
  --help_full                       [boolean]         Display the full detailed help message. 
  --show_hidden                     [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).

Typical pipeline command:

  nextflow run nf-core/drop -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.  
or `--helpFull`.  

Parameters defining the input and output options of the pipeline
  --input                              [string]           Path to the samplesheet file used by the pipeline. The file should be a TSV file. Equivalent to the 
`sampleAnnotation` parameter in the snakemake pipeline.  
  --outdir                             [string]           The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure. Equivalent to the `root` parameter in the snakemake pipeline.  
  --multiqc_title                      [string]           MultiQC report title. Printed as page header, used for filename if not otherwise specified. 
  --project_title                      [string]           Title of the project to be displayed on the rendered HTML output 
  --genome                             [string]           Equivalent to the `genomeAssembly` parameter in the snakemake pipeline. Either hg19/hs37d5 or 
hg38/GRCh38, depending on the genome assembly used for mapping  (accepted: hg19, hs37d5 , hg38, 
GRCh38)  
  --gene_annotation                    [string]           Path to GTF gene annotations file. Equivalent to the `geneAnnotation` parameter in the snakemake 
pipeline, with the difference that a file depicting the kev-value pairs is expected here instead. 
 
  --hpo_file                           [string]           Full path of the file containing HPO terms. If missing, it reads it from our webserver. Refer to 
[files-to-download](https://gagneurlab-drop.readthedocs.io/en/latest/prepare.html?highlight=drop_group#files-to-download). 
Equivalent to the `hpoFile` parameter in the snakemake pipeline.  
  --random_seed                        [boolean, integer] Specify a random seed for reproducibility 

Export counts options
  --ec_gene_annotations                [string] A comma-separated list of gene annotations to export. 
  --ec_exclude_groups                  [string] A comma-separated list of aberrant expression and aberrant splicing groups whose counts should not 
be exported. If empty, all groups are exported.  

Aberrant expression options
  --ae_skip                            [boolean] Skip aberrant expression analysis. 
  --ae_groups                          [string]  A comma-separated list of aberrant expression groups to run. If empty, all groups are run. 
  --ae_min_ids                         [integer] A positive number indicating the minimum number of samples that a group needs in order to be 
analyzed. We recommend at least 50. [default: 1]  
  --ae_fpkm_cutoff                     [integer] A positive number indicating the minimum FPKM value for a gene to be considered expressed. 
[default: 1]  
  --ae_implementation                  [string]  Methods to remove sample covariation in OUTRIDER.  (accepted: autoencoder, pca, peer) [default: 
autoencoder]  
  --ae_padj_cutoff                     [number]  A number between (0, 1] indicating the maximum FDR an event can have in order to be considered an 
outlier. [default: 0.05]  
  --ae_z_score_cutoff                  [number]  A non-negative number. Z scores (in absolute value) greater than this cutoff are considered as 
outliers. [default: 0]  
  --ae_max_tested_dimension_proportion [integer] An integer that controls the maximum value that the encoding dimension can take. [default: 3] 
 
  --ae_yield_size                      [integer] An integer that sets the batch size for counting reads within a bam file. If memory issues persist 
lower the yieldSize. [default: 2000000]  
  --ae_genes_to_test                   [string]  Full path to a yaml file specifying lists of candidate genes per sample to test during FDR 
correction. See the documentation for details on the structure of this file.  
  --ae_use_grid_search_to_obtain_q     [boolean] If true, the optimal latent space dimension for the autoencoder will be determined by grid search. 
If false, OHT will be performed..  

Aberrant splicing options
  --as_skip                            [boolean] Skip aberrant splicing analysis. 
  --as_groups                          [string]  A comma-separated list of aberrant splicing groups to run. If empty, all groups are run. 
  --as_min_ids                         [integer] A positive number indicating the minimum number of samples that a group needs in order to be 
analyzed. We recommend at least 50. [default: 1]  
  --as_recount                         [boolean] If true, it forces samples to be recounted. 
  --as_long_read                       [boolean] Set to true only if counting Nanopore or PacBio long reads. 
  --as_keep_non_standard_chrs          [boolean] Set to true if non standard chromosomes are to be kept for further analysis. 
  --as_skip_filter                     [boolean] If true, no filter is applied. We recommend filtering. 
  --as_min_expression_in_one_sample    [integer] The minimal read count in at least one sample required for an intron to pass the filter. 
[default: 20]  
  --as_quantile_min_expression         [integer] The minimum total read count (N) an intron needs to have at the specified quantile across samples to 
pass the filter. See --as_quantile_for_filtering. [default: 10]  
  --as_min_delta_psi                   [number]  The minimal variation (in delta psi) required for an intron to pass the filter. [default: 0.05] 
 
  --as_implementation                  [string]  Methods to remove sample covariation in FRASER.  (accepted: PCA, PCA-BB-Decoder) [default: PCA] 
 
  --as_fraser_version                  [string]  Version of FRASER to use. [default: FRASER2] 
  --as_delta_psi_cutoff                [number]  A non-negative number. Delta psi values greater than this cutoff are considered as outliers. 
Recommand 0.3 for FRASER, 0.1 for FRASER2. [default: 0.3]  
  --as_quantile_for_filtering          [number]  Defines at which percentile the as_quantile_min_expression filter is applied. Recommand 0.95 for 
FRASER, 0.75 for FRASER2. [default: 0.75]  
  --as_padj_cutoff                     [number]  Same as in aberrant expression. [default: 0.1] 
  --as_max_tested_dimension_proportion [integer] An integer that controls the maximum value that the encoding dimension can take. [default: 6] 
 
  --as_genes_to_test                   [string]  Full path to a yaml file specifying lists of candidate genes per sample to test during FDR 
correction. See the documentation for details on the structure of this file.  
  --as_use_grid_search_to_obtain_q     [boolean] If true, the optimal latent space dimension for the autoencoder will be determined by grid search. 
If false, OHT will be performed..  

Mono-allelic expression options
  --mae_skip                           [boolean] Skip mono-allelic expression analysis. 
  --mae_groups                         [string]  A comma-separated list of mono-allelic expression groups to run. If empty, all groups are run. 
 
  --ucsc_fasta                         [string]  Path to UCSC FASTA genome file. Equivalent to the `ucsc` option in the `genome` parameter in the 
snakemake pipeline.  
  --ucsc_fai                           [string]  Path to UCSC FASTA index file. 
  --ucsc_dict                          [string]  Path to UCSC sequence dictionary file. . If not provided, the pipeline will try to create it from 
the FASTA file.  
  --ncbi_fasta                         [string]  Path to NCBI FASTA genome file. Equivalent to the `ncbi` option in the `genome` parameter in the 
snakemake pipeline.  
  --ncbi_fai                           [string]  Path to NCBI FASTA index file. 
  --ncbi_dict                          [string]  Path to NCBI sequence dictionary file. If not provided, the pipeline will try to create it from the 
FASTA file.  
  --mae_gatk_header_check              [boolean] If true (recommended), it emits the header warnings of a VCF file when performing the allelic counts 
 
  --mae_padj_cutoff                    [number]  Same as in aberrant expression. [default: 0.05] 
  --mae_allelic_ratio_cutoff           [number]  A number between [0.5, 1) indicating the maximum allelic ratio allele1/(allele1+allele2) for the 
test to be significant. [default: 0.8]  
  --mae_add_af                         [boolean] Add the allele frequencies from gnomAD 
  --mae_max_af                         [number]  Maximum allele frequency (of the minor allele) cut-off. Variants with AF equal or below this number 
are considered rare. [default: 0.001]  
  --mae_max_var_freq_cohort            [number]  Maximum variant frequency among the cohort. [default: 0.05] 
  --mae_qc_vcf                         [string]  Full path to the vcf file used for VCF-BAM matching. Refer to 
[files-to-download](https://gagneurlab-drop.readthedocs.io/en/latest/prepare.html?highlight=drop_group#files-to-download). 
 
  --mae_qc_vcf_tbi                     [string]  Full path to the index of the vcf file used for VCF-BAM matching. Refer to 
[files-to-download](https://gagneurlab-drop.readthedocs.io/en/latest/prepare.html?highlight=drop_group#files-to-download) 
 
  --mae_qc_groups                      [string]  Same as 'groups', but for the VCF-BAM matching 
  --mae_dna_rna_match_cutoff           [number]  fraction (0-1) used to seperate 'matching' samples and 'non-matching' samples comparing the DNA and 
RNA data during QC. [default: 0.05]  

Generic options
  --email                              [string]          Email address for completion summary. 
  --multiqc_methods_description        [string]          Custom MultiQC yaml file containing HTML including a methods description. 
  --help                               [boolean, string] Display the help message. 
  --help_full                          [boolean]         Display the full detailed help message. 
  --show_hidden                        [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).

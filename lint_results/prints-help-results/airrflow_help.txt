help message of that parameter will be printed.  
or `--helpFull`.  

Input output options
  --input                       [string] Path to comma-separated file containing information about the samples in the experiment. 
  --mode                        [string] Specify the processing mode for the pipeline. Available options are "fastq" and "assembled".  
(accepted: fastq, assembled) [default: fastq]  
  --outdir                      [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                       [string] Email address for completion summary. 
  --miairr                      [string] Path to MiAIRR-BioSample mapping [default: 
${projectDir}/assets/reveal/mapping_MiAIRR_BioSample_v1.3.1.tsv]  

Protocol
  --library_generation_method   [string] Protocol used for the V(D)J amplicon sequencing library generation.  (accepted: 
specific_pcr_umi, specific_pcr, dt_5p_race, dt_5p_race_umi, sc_10x_genomics, trust4)  
  --race_linker                 [string] Path to fasta file containing the linker sequence, if no V-region primers were used but a linker 
sequence is present (e.g. 5' RACE SMARTer TAKARA protocol).  

Primer input and positions
  --vprimers                    [string]  Path to a fasta file containinc the V-region primer sequences. 
  --cprimers                    [string]  Path to a fasta file containing the C-region primer sequences. 
  --vprimer_start               [integer] Start position of V region primers (without counting the UMI barcode). [default: 0] 
  --cprimer_start               [integer] Start position of C region primers (without counting the UMI barcode). [default: 0] 
  --cprimer_position            [string]  Indicate if C region primers are in the R1 or R2 reads.  (accepted: R1, R2) [default: R1] 
  --primer_revpr                [boolean] Specify to match the tail-end of the sequence against the reverse complement of the primers. This 
also reverses the behavior of the --start argument, such that start position is relative to the 
tail-end of the sequence. (default: False)Maximum scoring error for the Presto MaxPrimer process for 
the C and/or V region primers identification.  

UMI barcode handling
  --umi_position                [string]  Indicate if UMI indices are recorded in the R1 (default) or R1 fastq file.  (accepted: R1, R2) 
[default: R1]  
  --umi_length                  [integer] UMI barcode length in nucleotides. Set to 0 if no UMIs present. [default: -1] 
  --umi_start                   [integer] UMI barcode start position in the index read. [default: 0] 
  --index_file                  [boolean] Indicate if UMI indices are recorded in a separate index file. 

Adapter trimming
  --trim_fastq                  [boolean] Whether to trim adapters in fastq reads with fastp. [default: true] 
  --adapter_fasta               [string]  Fasta file with adapter sequences to be trimmed. 
  --clip_r1                     [integer] Number of bases to clip 5' in R1 reads. [default: 0] 
  --clip_r2                     [integer] Number of bases to clip 5' in R2 reads. [default: 0] 
  --three_prime_clip_r1         [integer] Number of bases to clip 3' in R1 reads. [default: 0] 
  --three_prime_clip_r2         [integer] Number of bases to clip 3' in R2 reads. [default: 0] 
  --trim_nextseq                [boolean] Trim adapters specific for Nextseq sequencing 
  --save_trimmed                [boolean] Option to save trimmed reads. 

Sequence Assembly options
  --filterseq_q                 [integer] Quality threshold for pRESTO FilterSeq sequence filtering. [default: 20] 
  --primer_consensus            [number]  Maximum error for building the primer consensus in the pRESTO Buildconsensus step. [default: 
0.6]  
  --buildconsensus_maxerror     [number]  Maximum error for building the sequence consensus in the pRESTO BuildConsensus step. [default: 
0.1]  
  --buildconsensus_maxgap       [number]  Maximum gap for building the sequence consensus in the pRESTO BuildConsensus step. [default: 
0.5]  
  --cluster_sets                [boolean] Cluster sequences by similarity regardless of any annotation with pRESTO ClusterSets and annotate 
the cluster ID additionally to the UMI barcode. [default: true]  
  --primer_r1_maxerror          [number]  Maximum allowed error for R1 primer alignment. [default: 0.2] 
  --primer_r2_maxerror          [number]  Maximum allowed error for R2 primer alignment. [default: 0.2] 
  --maskprimers_align           [boolean] Align primers instead of scoring them. Used for protocols without primer fixed positions. 
  --primer_maxlen               [integer] Maximum allowed primer length when aligning the primers. [default: 50] 
  --primer_r1_mask_mode         [string]  Masking mode for R1 primers.  (accepted: cut, mask, trim, tag) [default: cut] 
  --primer_r2_mask_mode         [string]  Masking mode for R2 primers.  (accepted: cut, mask, trim, tag) [default: cut] 
  --maskprimers_align_race      [boolean] Use MaskPrimers align for a 5' RACE protocol. 
  --maskprimers_extract         [boolean] Use when primer sequences are unknown but when their approximate positions are known. 
  --primer_r1_extract_len       [integer] R1 primer extract length when using `--maskprimers_extract`. [default: 0] 
  --primer_r2_extract_len       [integer] R2 primer extract length when using `--maskprimers_extract`. [default: 0] 
  --assemblepairs_sequential    [boolean] Use AssemblePairs sequential instead of AssemblePairs align when assembling read pairs. 
  --align_cregion               [boolean] Align internal C-region for a more precise isotype characterization. 
  --internal_cregion_sequences  [string]  Provide internal C-region sequences for a more precise C-region characterization. Then also set the 
`align_cregion` flag.  
  --cregion_maxlen              [integer] Maximum allowed length when aligning the internal C-region. [default: 100] 
  --cregion_maxerror            [number]  Maximum allowed error when aligning the internal C-region. [default: 0.3] 
  --cregion_mask_mode           [string]  Mask mode for C-region alignment. [default: tag] 
  --skip_alignment_filter       [boolean] Skip filter step after alignment that ensures that locus should match the v_call chain, the sequence 
alignment should have at least 200 informative positions (excluding N or gaps), and maximum 10% N 
nucleotides in the alignment.  

Single cell analysis options
  --reference_10x               [string] Path to the reference directory required by cellranger. Can either be directory or tar.gz. 

Untargeted RNA-seq based analysis options
  --trust4_cell_barcode_read    [string] Specifies which read holds the barcodes  (accepted: R1, R2) 
  --trust4_umi_read             [string] Indicate if UMI indices are recorded in the R1 (default) or R2 fastq file.  (accepted: R1, R2) 
 
  --trust4_read_format          [string] Specifies where in the read the barcodes and UMIs can be found. 
  --trust4_barcode_whitelist    [string] path to the barcode whitelist. 

VDJ annotation options
  --reassign                    [boolean] Whether to reassign genes if the input file is an AIRR formatted tabulated file. [default: true] 
 
  --productive_only             [boolean] Subset to productive  sequences. [default: true] 
  --save_germlines              [boolean] Save germline database to reuse the cache in future runs. [default: true] 
  --reference_fasta             [string]  Path to the germline reference fasta. [default: 
https://raw.githubusercontent.com/nf-core/test-datasets/airrflow/database-cache/imgtdb_base.zip] 
 
  --reference_igblast           [string]  Path to the cached igblast database. [default: 
https://raw.githubusercontent.com/nf-core/test-datasets/airrflow/database-cache/igblast_base.zip] 
 
  --fetch_germlines             [string]  Fetch the selected germline reference bundle at runtime instead of using cached reference inputs. 
 (accepted: none, imgt, airrc-imgt) [default: none]  

Bulk filtering options
  --collapseby                  [string]  Name of the field used to collapse duplicated sequences. [default: sample_id] 
  --detect_contamination        [boolean] Whether to run the process to detect contamination. 
  --remove_chimeric             [boolean] Whether to apply the chimera removal filter. 

Clonal analysis options
  --clonal_threshold            [string, number] Set the clustering threshold Hamming distance value. Default: 'auto' [default: auto] 
  --lineage_trees               [boolean]        Perform clonal lineage tree analysis. 
  --cloneby                     [string]         Name of the field used to group data files to identify clones. [default: subject_id] 
  --crossby                     [string]         Name of the field used to identify external groups used to identify a clonal threshold. 
[default: subject_id]  
  --lineage_tree_builder        [string]         Lineage tree software to use to build trees within Dowser. If you change the default, also set the 
`lineage_tree_exec` parameter.  (accepted: raxml, igphyml) [default: raxml]  
  --lineage_tree_exec           [string]         Path to lineage tree building executable. [default: /usr/local/bin/raxml-ng] 
  --singlecell                  [string]         Name of the field used to determine if a sample is single cell sequencing or not. [default: 
single_cell]  
  --skip_all_clones_report      [boolean]        Skip report of EnchantR DefineClones for all samples together. 
  --skip_report_threshold       [boolean]        Skip report of EnchantR FindThreshold for all samples together. 
  --skip_clonal_analysis        [boolean]        Skip all clonal anlaysis processes 

Genotyping and Novel Alleles options
  --genotyping                  [boolean] Perform TIgGER novel allele and genotype inference. 
  --genotypeby                  [string]  Name of the field used to group data files to infer genotype. [default: subject_id] 
  --genotyping_clonal_threshold [number]  Threshold for determining if two sequences come from the same clone or not. [default: 0.2] 
  --single_clone_representative [boolean] Keep only one representative sequence per clone to perform genotype inference. 
  --novel_allele_inference      [boolean] Perform TIgGER novel allele inference. 

Translation and embedding options
  --translate                   [boolean] Generate a sequence amino acid translation with IgBlast. 
  --embeddings                  [string]  Generate sequence embeddings with amulety. 
  --embedding_chain             [string]  BCR or TCR chains to include for embedding. [default: H] 
  --use_gpu                     [boolean] Use GPU to generate embeddings. 

Report options
  --report_rmd                  [string]  Custom report Rmarkdown file. [default: ${projectDir}/assets/repertoire_comparison.Rmd] 
  --report_css                  [string]  Custom report style file in css format. [default: ${projectDir}/assets/nf-core_style.css] 
  --report_logo                 [string]  Custom logo for the report. [default: ${projectDir}/assets/nf-core-airrflow_logo_light.png] 
  --report_logo_img             [string]  Custom logo for the EnchantR reports. [default: 
${projectDir}/assets/nf-core-airrflow_logo_reports.png]  
  --skip_report                 [boolean] Skip repertoire analysis and report generation. 
  --skip_multiqc                [boolean] Skip multiqc report. 

Generic options
  --help                        [boolean, string] Display the help message. 
  --help_full                   [boolean]         Display the full detailed help message. 
  --show_hidden                 [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).

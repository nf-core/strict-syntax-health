Typical pipeline command:

  nextflow run nf-core/epitopeprediction -profile <docker/singularity/.../institute> --input samplesheet.csv --outdir <OUTDIR>

help message of that parameter will be printed.

Input/output options
  --input                       [string] Path to comma-separated file containing information about the samples in the experiment. 
  --outdir                      [string] The output directory where the results will be saved. You have to use absolute paths to storage on 
Cloud infrastructure.  
  --email                       [string] Email address for completion summary. 
  --multiqc_title               [string] MultiQC report title. Printed as page header, used for filename if not otherwise specified. 

Reference options
  --genome                      [string] Path to genome, that is the variant reference fasta. Can be compressed (.gz). 
  --genome_reference            [string] Specifies the genome reference (human: grch38, grch37, hg38, hg19; mouse: grcm39, grcm38, mm39, 
mm10) or an Ensembl URL. [default: grch38]  
  --proteome_reference          [string] Specifies the reference proteome fasta file that is used for self-filtering peptides derived from 
provided genomic variants.  

Peptide prediction options
  --biomart_dump_path           [string]  Specifies the path to a local Biomart dump (CSV/TSV) for offline usage of the pipeline. 
  --peptide_col_name            [string]  Specifies the column name in the input file that contains the peptide sequences. [default: 
sequence]  
  --min_peptide_length_classI   [integer] Specifies the minimum peptide length. [default: 8] 
  --max_peptide_length_classI   [integer] Specifies the maximum peptide length. [default: 12] 
  --min_peptide_length_classII  [integer] Specifies the minimum peptide length for MHC class II peptides. [default: 8] 
  --max_peptide_length_classII  [integer] Specifies the maximum peptide length for MHC class II peptides. [default: 25] 
  --tools                       [string]  Specifies the prediction tool(s) to use. [default: mhcnuggets] 
  --wild_type                   [boolean] Specifies whether wild-type sequences should be predicted. 
  --fasta_output                [boolean] Specifies that sequences of proteins, affected by provided variants, will be written to a FASTA 
file. [default: false]  
  --wide_format_output          [boolean] Specifies that the output file will be in wide format. [default: false] 
  --binder_only                 [boolean] Filter out non-binders from the final results. [default: false] 

Run optimisation
  --split_by_variants           [boolean] Split VCF file into multiple files by number of variants. 
  --peptides_split_maxchunks    [integer] Specifies the maximum number of peptide chunks. [default: 100] 
  --peptides_split_minchunksize [integer] Specifies the minimum number of peptides that should be written into one chunk. [default: 5000] 
 

External software
  --netmhc_system               [string] Specifies the operating system in use (Linux or Darwin). This is only necessary if conda is used. 
 (accepted: linux, darwin) [default: linux]  
  --netmhcpan_path              [string] To use the 'netmhcpan' tool, specify the path to the original software tarball for NetMHCpan 4.0 
here.  
  --netmhciipan_path            [string] To use the 'netmhciipan' tool, specify the path to the original software tarball for NetMHCIIpan 3.1 
here.  

Generic options
  --multiqc_methods_description [string]          Custom MultiQC yaml file containing HTML including a methods description. 
  --help                        [boolean, string] Display the help message. 
  --help_full                   [boolean]         Display the full detailed help message. 
  --show_hidden                 [boolean]         Display hidden parameters in the help message (only works when --help or --help_full are provided).
